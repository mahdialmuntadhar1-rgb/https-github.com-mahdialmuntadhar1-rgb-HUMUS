# 401 Unauthorized Fix

## Problem
Frontend requests to `/rest/v1/businesses` and `/rest/v1/posts` returning 401 in production.

## Root Cause
RLS policies blocking public access - need explicit public SELECT policies.

## Solution

### 1. Run SQL Policy Fix
Execute in Supabase SQL Editor:
```sql
-- Copy contents of supabase/fix_401_errors.sql
```

### 2. Frontend Env Var Validation
Updated `src/lib/supabaseClient.ts`:
- ✅ Validates `VITE_SUPABASE_URL` exists
- ✅ Validates `VITE_SUPABASE_ANON_KEY` exists  
- ✅ Throws error if missing
- ✅ Logs connection details for debugging

### 3. Vercel Environment Variables
Ensure these are set in Vercel:
```
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
```

### 4. RLS Policies Applied
**Public Access (No Auth Required):**
- `businesses_public_select` - Anyone can read businesses
- `posts_public_select` - Anyone can read posts

**Authenticated Access:**
- `businesses_authenticated_insert` - Logged in users can submit
- `posts_authenticated_insert` - Logged in users can post
- Owner update/delete policies for both tables

## Verification Steps
